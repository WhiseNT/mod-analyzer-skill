# mod-analyzer-skill

用于分析 Minecraft Mod / 模组的 Claude 技能。

## 概述

本技能将 Minecraft Mod 的 jar / 源码 / 数据包资源转译成玩家与策划能读懂的文档。支持三种分析类型：

- **游戏内容文档**（默认）— 面向玩家/策划，回答"这个 mod 怎么玩"
- **设计分析报告** — 面向策划/整合包作者，回答"这个 mod 设计得好不好"
- **代码分析报告** — 面向开发者/魔改作者，回答"这个 mod 代码怎么组织的、怎么改"


## 三种分析类型

| 类型 | 输出产物 | 目标受众 | 核心问题 |
|------|----------|----------|----------|
| **游戏内容文档**（默认） | `模组游戏内容文档-<mod-name>.md` | 玩家、策划 | 这个 mod 怎么玩？ |
| **设计分析报告** | `模组设计分析报告-<mod-name>.md` | 策划、整合包作者 | 这个 mod 设计得好不好？ |
| **代码分析报告** | `模组代码分析报告-<mod-name>.md` | 开发者、魔改作者 | 这个 mod 代码怎么组织的？怎么改？ |

三种分析共享阶段 0~2（输入确认 + 静态清单 + 反编译/源码读取），阶段 3 开始分岔。

## 工作流

### 基础阶段（所有分析类型共享）

### 基础阶段（所有分析类型共享）

1. **阶段 0：确定输入与边界** — 确认 jar/源码/版本/loader/输出深度
2. **阶段 1：静态清单盘点** — 运行 `inspect_mod_jar.py` 建立全局地图
3. **阶段 2：反编译或读取源码** — 使用 Vineflower/CFR 或直接读源码

### 分支 A：游戏内容文档（默认）

4. **阶段 3：建立内容清单** — 将代码与资源归并为玩家可理解的内容模块
5. **阶段 4：抽取核心玩法循环** — 从玩家反复做什么出发定义循环
6. **阶段 5：细粒度流程追踪** — 至少 3 条流程（新手入口/核心循环/复杂隐藏）
7. **阶段 6：输出模组游戏内容文档** — 按 `references/report-template.md` 结构输出

### 分支 B：设计分析报告

4. **完成基础内容清单和玩法循环**（阶段 3~6 的基础部分）
5. **按 `references/design-analysis-template.md` 输出报告**
   - 评价设计目标、核心循环完整性、新手引导、内容组织、难度曲线
   - 按 80 分制评分
   - 所有评分必须有证据支撑

### 分支 C：代码分析报告

4. **确保静态清单和反编译完整**
5. **按 `references/code-analysis-template.md` 输出报告**
   - 梳理注册架构、Mixin、事件、网络包、配置、数据驱动内容
   - 提供魔改指导（KubeJS/CraftTweaker 示例）
   - 评估可魔改友好度

## 使用场景

- 分析 Minecraft Mod、mod jar、整合包里的某个 mod
- 反编译 Forge/NeoForge/Fabric/Quilt mod 并理解玩法
- 从源码或 jar 中梳理物品、方块、配方、世界生成、生物、维度、GUI 等
- 输出模组简介、主要玩法、核心玩法循环、细粒度玩家流程、内容设计文档
- **模组设计分析**：评价玩法循环、难度曲线、新手引导、内容组织的设计质量
- **模组代码分析**：面向开发/魔改的注册架构、mixin、事件、API、配置项、魔改点梳理

## 绝对原则

- 先证据，后结论
- 不要运行不可信 jar
- 不要只凭名称猜玩法
- 数据驱动和代码驱动同等重要
- 把代码路径翻译成玩家路径
- 不确定就标注不确定

## 依赖

- Python 3
- Java Runtime（用于运行 Java 反编译器）
- 推荐：Vineflower / CFR / FernFlower 反编译器 jar
